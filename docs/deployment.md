# Deployment Guide

## Prerequisites

- Docker 24+ and Docker Compose v2
- 4GB+ RAM (8GB recommended)
- 20GB+ disk space
- Open ports: 3000 (web), 8000 (API), 9000-9001 (MinIO)

## Quick Start (Single Machine)

```bash
git clone https://github.com/your-org/white-ops.git
cd white-ops
./scripts/setup.sh
```

This automatically:
1. Generates cryptographically secure secrets in `.env`
2. Builds all Docker images
3. Starts all 7 services
4. Creates the admin account

Access: `http://localhost:3000`

## Manual Setup

```bash
cp .env.example .env
# Edit .env - set all passwords and at least one LLM API key
make build
make up
```

## Configuration

Edit `.env` before starting:

| Variable | Required | Description |
|----------|----------|-------------|
| `SECRET_KEY` | Yes | App secret (auto-generated by setup.sh) |
| `JWT_SECRET_KEY` | Yes | JWT signing key |
| `POSTGRES_PASSWORD` | Yes | Database password |
| `REDIS_PASSWORD` | Yes | Redis password |
| `MINIO_ROOT_PASSWORD` | Yes | File storage password (min 8 chars) |
| `ANTHROPIC_API_KEY` | At least one LLM | Claude API key |
| `OPENAI_API_KEY` | At least one LLM | GPT API key |
| `OLLAMA_BASE_URL` | For local LLM | Ollama endpoint |
| `ADMIN_EMAIL` | Yes | First admin account email |
| `ADMIN_PASSWORD` | Yes | First admin account password |

## Adding Worker Nodes

On each additional PC:

```bash
# Option 1: Script
./scripts/add-worker.sh <MASTER_IP>

# Option 2: Manual
# Copy docker-compose.worker.yml to remote PC
# Set MASTER_URL, REDIS_PASSWORD, MINIO_ROOT_PASSWORD
# docker compose up -d
```

Then approve the worker in Admin Panel > Workers.

## Production Checklist

- [ ] All passwords are strong and unique (not defaults)
- [ ] At least one LLM API key configured
- [ ] CORS_ORIGINS set to your domain (not localhost)
- [ ] TLS/HTTPS configured (reverse proxy or Let's Encrypt)
- [ ] Firewall rules: only 3000 and 8000 exposed externally
- [ ] Database backups configured (pg_dump cron)
- [ ] Log rotation configured
- [ ] Monitoring enabled (`docker compose --profile monitoring up -d`)
- [ ] Rate limiting verified
- [ ] Admin password changed from default

## Monitoring

```bash
# Enable Prometheus + Grafana
docker compose --profile monitoring up -d

# Access
# Prometheus: http://localhost:9090
# Grafana:    http://localhost:3001 (admin / see GRAFANA_ADMIN_PASSWORD)
```

## Backup & Restore

```bash
# Backup database
docker compose exec postgres pg_dump -U whiteops whiteops > backup.sql

# Restore
docker compose exec -i postgres psql -U whiteops whiteops < backup.sql

# Backup files (MinIO)
docker compose exec minio mc mirror /data /backup
```

## Updating

```bash
git pull
make build
make up  # Rolling restart
```

## Troubleshooting

```bash
make status        # Check all services
make logs          # View all logs
make logs-server   # View server logs only
make shell-server  # Open shell in server container
make shell-db      # Open psql shell
```

Common issues:
- **Port conflict**: Change ports in `.env` (SERVER_PORT, WEB_PORT)
- **Worker can't connect**: Check firewall, ensure ports 8000/6379/9000 accessible
- **LLM errors**: Verify API key in Settings page
- **Out of memory**: Increase Docker memory limit or reduce WORKER_MAX_AGENTS
